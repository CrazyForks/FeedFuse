# 使用指南

这份文档面向已经运行 FeedFuse 的使用者，帮助你完成首次登录、账号管理和日常阅读配置。

如果还没有启动服务，请先看 [部署指南](./deploy.md)。

## 1. 首次登录

打开部署后的访问地址，进入登录页：

```text
http://127.0.0.1:9559/login
```

全新数据库首次登录使用：

- 用户名：`admin`
- 密码：`.env` 里的 `AUTH_INITIAL_PASSWORD`

首次登录成功后，初始密码会写入数据库。之后继续使用应用内保存的账号密码登录，而不是重新读取 `.env` 里的旧值。

## 2. 修改当前账号

登录后打开：

```text
设置中心 -> 账号与安全 -> 当前账号 -> 编辑
```

你可以在这里修改：

- 用户名
- 新密码

只修改用户名不会让当前会话失效。修改密码后，系统会刷新当前会话。

## 3. 管理用户

只有 `管理员` 可以看到 `用户管理`。

管理员可以：

- 新增用户
- 修改非初始用户的用户名
- 调整用户角色为 `管理员` 或 `成员`
- 启用或禁用用户

只有 `初始用户` 可以删除其他用户。初始用户固定是系统创建的第一个用户，即首次登录时的 `admin` 账号。初始用户可以改名，但仍保留初始用户权限。

删除用户会同时删除该用户拥有的数据，包括订阅源、分类、Fever 服务、AI 解读任务和系统日志关联数据。这个操作无法恢复。

## 4. 理解角色

FeedFuse 当前有两种角色：

- `管理员`：可以进入用户管理，创建、编辑、启用或禁用用户
- `成员`：只能管理自己的订阅、设置和账号资料

所有用户都可以修改自己的用户名和密码。

## 5. 数据隔离

同一个 FeedFuse 实例可以给多个用户使用。不同用户之间默认隔离：

- RSS 源和分类
- 文章阅读状态
- Fever 服务和同步状态
- AI 配置、翻译配置和 UI 设置
- AI 解读、摘要、翻译和全文抓取任务

不同用户可以添加相同的 RSS 地址或 Fever 服务账号。它们会作为各自账号下的独立数据保存。

## 6. 添加 RSS 源

在主界面点击添加订阅源，填写 RSS 地址并选择分类。

如果需要批量迁移，可以使用：

```text
设置中心 -> RSS -> OPML 导入
```

导出的 OPML 只包含当前登录用户自己的订阅源。

## 7. 配置 Fever 服务

如果你使用 FreshRSS、Tiny Tiny RSS 等兼容 Fever API 的服务，打开：

```text
设置中心 -> Fever 账号
```

你可以添加 Fever 服务、手动同步、启用或停用服务，并配置自动同步间隔。

Fever 托管源由远端服务负责管理。FeedFuse 会在本地展示这些源和文章，但不建议从普通 RSS 源编辑入口修改 Fever 托管源。

## 8. 配置 AI

如果只阅读 RSS，可以先跳过 AI 配置。

需要摘要、翻译或 `AI解读` 时，打开：

```text
设置中心 -> AI
```

填写模型、API 地址和 API 密钥。兼容 OpenAI API 的服务通常也可以使用，只要提供服务商给出的 `/v1` 地址和密钥。

翻译默认复用主 AI 配置。只有翻译要走另一套服务时，才切换为单独配置。

## 9. 退出登录

打开：

```text
设置中心 -> 账号与安全 -> 退出登录
```

退出后会结束当前会话并返回登录页。

# 前端类型安全

## 类型来源

- 跨层实体优先定义在 `src/types/index.ts`
- API 请求和响应类型由 `src/lib/api/apiClient.ts` 暴露，组件不重复声明近似 DTO
- feature 私有表单与视图模型保留在对应 feature 内

## 边界规则

- 外部输入在 API route 使用 Zod 或现有 schema 校验；前端类型不能替代运行时校验
- 可空、可选和状态联合类型必须反映后端真实响应，不用类型断言掩盖缺失字段
- 新增响应字段时同步修改 route、`apiClient`、共享类型及前端消费测试
- 权限和用户类型使用后端返回的稳定字段，不在组件中通过用户名或可变属性推导
